# Свободные аудитории

Проект нужен для страницы, которая показывает свободные аудитории на текущую пару.
Данные берутся напрямую с официального сайта расписания `schedule.siriusuniversity.ru` с помощью livewire.

## Что уже выяснено

Официальный сайт использует Livewire. Для аудиторий есть отдельный раздел:

```text
https://schedule.siriusuniversity.ru/classroom
```

Livewire endpoint для этого раздела:

```text
https://schedule.siriusuniversity.ru/livewire/message/classroom.classroom-main-grid
```

Компонент в `fingerprint.name`:

```text
classroom.classroom-main-grid
```

Начальное состояние лежит в HTML страницы в атрибуте:

```text
wire:initial-data="..."
```

Для POST-запроса также нужен токен из HTML:

```text
window.livewire_token = '...';
```

## Важные поля Livewire

В `serverMemo.data` у компонента аудиторий есть такие поля:

```text
search          строка поиска по аудиториям
classroom       выбранная аудитория
classroomsList  список найденных аудиторий
events          расписание выбранной аудитории
count           количество занятий в выбранной аудитории на неделе
date            дата начала/текущая дата отображаемой недели
numWeek         номер недели
addNumWeek      смещение недели вперед
minusNumWeek    смещение недели назад
```

`classroomsList` показывает только 20 элементов, но меняется при изменении
`search`. Значит полный список аудиторий нужно собирать несколькими поисковыми
запросами и объединять результаты.

Примеры найденных поисков:

```text
search = ""     -> первые 20 аудиторий
search = "К_"   -> кабинеты колледжа: 1,26К_9, 1,27К_8, 1.25К_10 ...
search = "1."   -> аудитории с 1.: 1.36К_0, 1.19..., Лаборатория 1.8.20 ...
search = "Аль"  -> аудитории Альфа
```

## Как искать аудитории

Нужно открыть `/classroom`, достать `wire:initial-data`, а затем отправлять
Livewire update, который меняет поле `search`.

Вероятный update для поиска:

```json
{
  "type": "syncInput",
  "payload": {
    "id": "<component_id>",
    "name": "search",
    "value": "К_"
  }
}
```

Если `syncInput` не сработает, использовать вариант через `$set`:

```json
{
  "type": "callMethod",
  "payload": {
    "id": "<component_id>",
    "method": "$set",
    "params": ["search", "К_"]
  }
}
```

Из ответа нужно брать:

```text
serverMemo.data.classroomsList
```

Livewire иногда возвращает `classroomsList` как массив, а иногда как объект с
числовыми ключами. В коде нужно поддержать оба варианта.

Минимальный набор поисковых строк для сбора списка:

```python
SEARCH_TERMS = [
    "",
    "К_",
    "1.",
    "1,",
    "2.",
    "Аль",
    "Альфа",
    "Бета",
    "ЛК",
    "Газ",
    "Рос",
    "Лаб",
    "дистан",
]
```

Этот список не гарантирует 100% полноту, но уже находил 76 аудиторий. Позже его
можно расширить цифрами, буквами или ручными префиксами.

## Как получить расписание аудитории

Когда на сайте выбирают аудиторию, Livewire вызывает метод `set`:

```json
{
  "type": "callMethod",
  "payload": {
    "id": "<component_id>",
    "method": "set",
    "params": ["1.33К_2 (Основной)"]
  }
}
```

В ответе приходит:

```text
serverMemo.data.classroom = "1.33К_2 (Основной)"
serverMemo.data.count = 24
serverMemo.data.events = {...}
```

`events` - это расписание выбранной аудитории. Оно приходит не плоским списком,
а словарем, где ключи вроде `3_0`, `4_1`, `2_5` являются внутренними координатами
таблицы расписания.

Пример структуры:

```json
{
  "3_0": [
    {
      "date": "27.04.2026",
      "dayWeek": "ПН",
      "startTime": "13:30",
      "endTime": "14:50",
      "discipline": "Элементы высшей математики",
      "group": "К0709-24/2",
      "classroom": "1.33К_2 (Основной)"
    }
  ]
}
```

Для приложения ключи `3_0`, `4_1` не важны. Нужно пройти по всем значениям
словаря `events`, собрать все вложенные занятия в один список и дальше работать
с обычным списком событий.

## Как понять, свободна аудитория или нет

У пары есть фиксированный интервал. Сетка пар:

```text
08:45-10:05
10:20-11:40
11:55-13:15
13:30-14:50
15:05-16:25
16:40-18:00
18:15-19:35
19:50-21:10
```

Аудитория занята, если на выбранную дату есть занятие, время которого пересекается
с текущей парой:

```text
event.startTime < slot.end and event.endTime > slot.start
```

Например, для даты `30.04.2026` и пары `13:30-14:50` нужно искать события:

```text
event.date == "30.04.2026"
event.startTime < "14:50"
event.endTime > "13:30"
```

Если таких событий нет, аудитория свободна.

## Алгоритм текущей реализации

1. Открыть `https://schedule.siriusuniversity.ru/classroom`.
2. Достать `wire:initial-data`, `fingerprint`, `serverMemo` и Livewire token.
3. Собрать список аудиторий через несколько значений `search`.
4. Убрать дубликаты аудиторий.
5. Для каждой аудитории вызвать Livewire `set("<classroom>")`.
6. Из ответа взять `serverMemo.data.events`.
7. Превратить `events` в плоский список занятий.
8. Проверить пересечение занятий с текущей датой и текущей парой.
9. Вернуть два списка: свободные аудитории и занятые аудитории с причиной занятости.

## Что важно для реализации

- Нужно использовать один `opener` с cookies: сначала GET `/classroom`, затем POST
  в Livewire endpoint.
- Список аудиторий стоит кешировать локально на 10-30 минут, потому что сбор через
  много поисковых запросов может быть медленным.
- Расписание конкретной аудитории тоже можно кешировать на несколько минут.
- Если официальный сайт недоступен, в UI нельзя показывать аудитории свободными:
  нужно показывать ошибку проверки.

## Основные функции в `app.py`

```text
OfficialClassroomClient._get_initial_state()   получить token, fingerprint, serverMemo
OfficialClassroomClient._post_livewire(...)    отправить Livewire POST
OfficialClassroomClient.search_classrooms(...) получить classroomsList для поисковой строки
cached_classrooms()                            собрать и закешировать уникальный список аудиторий
OfficialClassroomClient.fetch_schedule(...)    получить events аудитории через method=set
OfficialClassroomClient._flatten_events(...)   превратить events в обычный список
current_lesson_slot()                          определить текущую пару по времени
event_overlaps(...)                            проверить пересечение занятия с парой
get_free_classrooms()                          собрать итоговый список свободных аудиторий
```

## Текущий статус проекта

В проекте есть Flask-страница и API `/api/free-classrooms`.

При пустом поле аудиторий приложение:

1. собирает список аудиторий через `search`;
2. получает расписание каждой аудитории через официальный Livewire `set`;
3. проверяет текущую дату и текущий слот пары;
4. возвращает свободные и занятые аудитории.

Проверка `30.04.2026 13:30-14:50` через официальный сайт нашла 75 аудиторий:
57 свободных и 18 занятых.
